The Big Picture

Digital systems convert physical signals (temperature, light, sound) into binary values, process them with logic gates and state machines, store results in memory, and convert back to analog outputs. This guide walks through each layer:

  • Layers 1–2: Power and bits — the physical foundation.
  • Layers 3–5: Gates, Boolean algebra, and minimization — how to design efficient combinational logic.
  • Layers 6–7: Flip-flops, counters, and state machines — how to add memory and time-dependent behavior.
  • Layers 8–9: ADC/DAC and memory hierarchy — how to connect the digital world to analog reality and how processors work at the architectural level.
Embedded Systems Foundation

Power & Electricity in Digital Circuits

Before digital logic, understand the physical layer — voltage, current, and power govern every embedded decision, and the rest of the semester builds on that foundation.

🔗 This section underpins every section that follows — TTL voltage levels, pull-up resistors, and power dissipation appear throughout the course.
Semester Roadmap: The course starts with analog versus digital behavior, moves into binary number systems and logic gates, then uses Boolean algebra and Karnaugh maps to simplify circuits. From there it builds memory elements, counters, and state machines, and finishes with ADC/DAC conversion plus memory and processor architecture. Each section depends on the prior ones — if something feels unclear, revisit earlier sections first.

Ohm's Law & Kirchhoff's Laws

  • V = IR — Voltage equals current × resistance.
  • KVL: Sum of voltages around any closed loop = 0.
  • KCL: Sum of currents entering a node = sum leaving.
  • Embedded relevance: Pull-up resistors, current-limiting for LEDs, and bus termination all apply KVL/KCL directly.

Power in Digital Circuits

  • P = IV = V²/R = I²R
  • Digital ICs dissipate power in two forms: static (leakage) and dynamic (switching).
  • Dynamic power: Pdyn = α·C·VDD²·f — where α = activity factor, C = capacitance, f = clock frequency.
  • Reducing VDD is the most effective power-saving technique (quadratic effect).

Voltage Levels & Logic Standards

  • TTL (5V): LOW = 0–1V, HIGH = 2.5–5V.
  • CMOS (3.3V / 1.8V): Common in modern embedded systems.
  • Voltage margins matter — noise can corrupt logic if margins are thin.
  • Never directly connect TTL output to CMOS input without level-shifting.

Pull-up & Pull-down Resistors

  • Pull-up: Connects pin to VCC; forces HIGH when pin floats.
  • Pull-down: Connects pin to GND; forces LOW when pin floats.
  • With switch open and 1kΩ pull-up to +5V: I = 0, so Vpin = +5V (HIGH).
  • With switch closed: Vpin = 0V, I = 5mA through resistor.
  • Critical for embedded I²C, SPI, and GPIO design.

Decoupling Capacitors

  • Placed close to IC power pins (VCC to GND).
  • Supply instantaneous current during switching transients.
  • Typical values: 100nF ceramic + 10µF bulk per IC.
  • Absence causes supply noise → logic glitches in embedded systems.

Ground Planes & Signal Integrity

  • Solid ground plane reduces impedance and EMI.
  • Ground bounce: Simultaneous switching outputs (SSO) cause transient ground shifts.
  • Keep high-speed digital traces short and impedance-controlled (typically 50Ω).
  • Separate analog and digital ground planes; join at single point.

Chapter 1

Binary Concepts & Number Systems

All digital computation reduces to two voltage states. Mastering number representations is the foundation of every circuit design decision.

🔗 Binary arithmetic feeds directly into ALU design, memory addressing, and ADC/DAC resolution (Section 8).

Why Binary?

  • Transistors operate as switches — two stable states: ON / OFF.
  • Noise immunity: two states are far enough apart to withstand voltage perturbations.
  • Benefits: storage, perfect reproduction, flawless transmission.
  • Trade-off: analog signal needs multiple bits (≥10 bits for 0.1% accuracy) → more transmission lines.

Binary Logic States

BooleanAlgebra+True Voltage−True Voltage
True1HIGH (H)LOW (L)
False0LOW (L)HIGH (H)

Positive-true convention: HIGH = True = 1. This is standard in most embedded systems. When a pin is at +5V or +3.3V (depending on the voltage standard), it reads as a logic 1.
Negative-true convention: bar over label (e.g. S̄), LOW = True. Used in some legacy circuits and when signal polarity is inverted (e.g., active-low enable pins). When the pin is at 0V, the signal is "true" or "active." This matters in practice: an active-low reset pin means the circuit resets when you pull it to 0V, not when you pull it to 1V.

Reading concern: Always check schematics for bar notation. Confusing positive and negative logic is a common source of embedded firmware bugs — your code may be trying to set a pin HIGH when it should go LOW to activate a feature.

Unsigned Binary

  • Positional notation: each bit represents a power of 2. The rightmost bit is 2⁰, the next is 2¹, then 2², and so on.
  • Example: 01001111₂ = 0·128 + 1·64 + 0·32 + 0·16 + 1·8 + 1·4 + 1·2 + 1·1 = 64+8+4+2+1 = 79₁₀
  • Max value for n bits: 2ⁿ − 1. For 8 bits: 255. For 16 bits: 65,535. This is why ADC resolution matters — a 12-bit ADC can represent 4,096 distinct values.
  • Conversion: decimal→binary by repeatedly dividing by 2 and collecting remainders in reverse order. For example, 79÷2=39r1, 39÷2=19r1, 19÷2=9r1, etc. Reading remainders backward gives 01001111.
  • Bit significance: MSB (Most Significant Bit) = leftmost = highest power; LSB (Least Significant Bit) = rightmost = 2⁰ = 1's place.

Reading concern: Overflow is silent in unsigned arithmetic. Adding 1 to 255 in 8-bit unsigned wraps to 0. Many firmware bugs come from unsigned integer overflow — your counter hits 256 but wraps to 0. Always be aware of your bit width and maximum representable value.

BCD — Binary Coded Decimal

  • Each decimal digit encoded separately in 4 bits.
  • 79 → 7=0111, 9=1001 → 01111001 (BCD) ≠ 01001111 (unsigned)
  • Useful for decimal displays (7-segment, LCDs).
  • Wastes bit combinations (1010–1111 unused per nibble).

Negative Numbers: 2's Complement

  • Sign-magnitude approach: Use the MSB as a sign bit (0=positive, 1=negative) and the remaining bits for magnitude. Intuitive but problematic: you have two representations of zero (+0 and −0), and addition doesn't work correctly in hardware.
  • 2's Complement (the standard): Solves the problem by making the MSB have a negative weight. The insight: N + (−N) should equal 0 in modular arithmetic. So instead of a separate sign bit, we represent negative numbers such that addition works naturally with overflow discarded.
  • How to compute: To negate N: flip all bits (one's complement), then add 1. Alternatively, −N = −2MSB + remaining bits. For an 8-bit number, the MSB represents −128.
  • 4-bit example: Range is −8 to +7. Positive 5 = 0101. To get −5: flip bits (1010), add 1 (1011). Verify: 0101 + 1011 = 10000 (5-bit result), discard the overflow → 0000 in 4-bit space, which is 0. Correct!
  • Why it matters: Both addition and multiplication work correctly in 2's complement without special handling. Your CPU's ALU doesn't need separate unsigned and signed circuits; the same hardware works for both.

Reading concern: The MSB is special in 2's complement — it has a negative weight. Don't mix signed and unsigned: treating −5 (1011) as unsigned 11 is a logic error. Always be explicit about whether a variable is signed or unsigned in your firmware.

Hexadecimal

  • Base-16 notation: Groups every 4 bits into one symbol (0–9, A–F). This is the "sweet spot" for humans — binary is too verbose, decimal doesn't align with bit positions.
  • Conversion rule: Each hex digit is exactly 4 binary bits. 0x4F means 0100 1111 in binary. Every hex digit is independent — no borrowing or carrying across digit boundaries when converting.
  • 79₁₀ example: In binary: 01001111. Group from right: 0100 1111. Convert each group: 0100=4, 1111=F. Result: 0x4F.
  • Embedded ubiquity: Memory addresses, register values, and hardware configuration constants are always shown in hex. Example: 0xFF (all bits set to 1), 0x1A3F (specific register configuration), 0x8000 (bit 15 set). Your embedded IDE likely shows memory dumps in hex.
  • Quick mental math: 0x100 = 256 (2⁸), 0x1000 = 4096 (2¹²), 0x10000 = 65536 (2¹⁶). Learning powers of 16 is essential for embedded work.

Chapter 2

Boolean Algebra & DeMorgan's Theorems

Boolean algebra is the bridge between truth tables and real gate networks. It provides the mathematical framework to analyze, simplify, and transform logic expressions before hardware implementation.

🔗 Boolean identities enable algebraic minimization (Section 4); DeMorgan's enables gate substitution crucial in NAND-only designs.

Core Boolean Identities

IdentityExpression
IdempotentAA = A ; A+A = A
ComplementA·A = 0 ; A+A = 1
Double negationA = A
AnnulmentA·0 = 0 ; A+1 = 1
IdentityA·1 = A ; A+0 = A
Absorption A + AB = A
Reason: A + AB = A(1+B) = A·1 = A — if A is true the sum is true; if A is false then AB is false.
A(A + B) = A
Reason: A(A + B) = A·1 = A — A absorbs the larger expression.
Distributive A(B + C) = AB + AC
Also useful: A + BC = (A + B)(A + C) — OR distributes over AND in this form.

How to Read Boolean Expressions

  • Order of operations: Parentheses first, then NOT, then AND, then OR.
  • Example: A + BC means "A OR (B AND C)" — the AND happens before the OR.
  • Why identities matter: Use them to remove duplicate terms and unnecessary gates. The same logic can be written many ways; simplification yields fewer transistors and lower power.
  • Positive vs negative logic: Positive logic treats 1 as true. Negative logic flips the meanings of 1 and 0, which is why the same circuit looks different in different notations.

DeMorgan's Theorems

Two critical identities for circuit transformation:

Theorem 1: A+B = A·B
Theorem 2: A·B = A+B

Rule: Change gate shape (AND↔OR) and invert all inputs and outputs.

NAND symbol with inverted inputs ≡ OR gate.
NOR symbol with inverted inputs ≡ AND gate.

Worked: XOR via DeMorgan

Start: A ⊕ B = AB + AB

Add AA̅=0, BB̅=0:

= A(A+B) + B(A+B)

Apply DeMorgan: (A+B) = AB

= A·AB + B·AB

Apply DeMorgan again → 4-NAND XOR.

Embedded Systems Application

  • FPGA synthesis tools apply Boolean minimization automatically.
  • Manual simplification reduces gate count → lower power, smaller die.
  • DeMorgan transforms NOR-based logic (preferred in some processes) to equivalent AND/OR forms.
  • Firmware bitmask operations use Boolean: reg &= ~(1<<n) = clear bit n.

Chapter 2

Logic Minimization & Karnaugh Maps

Raw truth-table expressions are inefficient. Minimization yields the simplest equivalent gate network — fewer gates means lower cost, power, and propagation delay.

🔗 Minimized expressions feed directly into state machine logic design (Sections 7 & 6) and adder circuits.
Why minimize? A truth table produces expressions that are correct but wasteful. For example, a 4-input function might need 8 AND gates in "sum of products" form, but careful grouping might reduce it to 2 gates. This matters enormously: fewer gates = lower power consumption, smaller chip area, and faster propagation delay. A 50% gate reduction on a million-transistor chip saves gigawatts at data center scale. Karnaugh maps let you find the best minimization by hand for functions up to 4 variables; beyond that, automated tools (Espresso, Quine-McCluskey) take over.

Two Minimization Strategies

  1. Algebraic: Apply Boolean identities. Suitable for any number of variables; can use software (Espresso, Quine-McCluskey).
  2. K-map: Visual grouping method. Efficient and intuitive for ≤4 inputs. Limited to ≤4 variables by hand.

K-Map Rules

  1. Arrange input combinations in Gray code order (one bit changes at a time).
  2. Group 1s into rectangles of size 1, 2, 4, or 8 (powers of 2).
  3. Larger groups → simpler (fewer literal) expressions.
  4. Groups may overlap and wrap around edges.
  5. OR all group expressions for the final SOP expression.
  6. Alternatively, group 0s to get expression for Q.

Example: 2-variable K-Map

Truth table: A=0,B=0→1; A=0,B=1→1; A=1,B=0→0; A=1,B=1→1

Chapter 3

Flip-Flops & Sequential Logic

Flip-flops introduce memory — the circuit's output depends on both current input AND past state. This transitions circuits from combinational to sequential.

🔗 Flip-flops are the elementary unit of registers (Section 6), counters (Section 6), and state machines (Section 7). Every synchronous embedded system depends on them.
The memory breakthrough: So far (Sections 1–4) every circuit's output depends only on its current input — these are combinational circuits. A flip-flop is a feedback loop of gates that remembers a 1-bit value until an external signal (like a clock edge) changes it. This memory is the foundation of registers (multi-bit storage), counters (sequences of states), and entire computers. Without flip-flops, digital systems would have no way to retain past values, and computation would be impossible.

SR Latch (NOR-based)

Structure: Two cross-coupled NOR gates. The output of each gate feeds one input of the other, creating a feedback loop that holds state.

SRQState
101Set (Q=1)
010Reset (Q=0)
00QprevHold/Memory
11—AVOID

Why S=R=1 is forbidden: When both inputs are 1, both NOR outputs are forced to 0, so Q and Q̅ are identical. This violates the assumption that they are opposites. When S and R then return to 0, the latch state becomes undefined — race conditions cause Q to randomly settle to 0 or 1.

Reading concern: The SR latch is asynchronous (responds immediately to input changes). Never let S and R both go high in real circuits, or the outcome is unpredictable. Always think: "What happens if both signals are high?" This is a design flaw that some logic must prevent.

Debounced Switch (SR Application)

Mechanical switches bounce during contact → unintended multi-transitions.

SR latch with switch at A or B: bounce causes A=B=1 → "retain previous" state → Q unchanged → counter sees single clean pulse.

Embedded: every physical button/switch should be debounced — hardware SR or software delay.

Clocked SR Flip-Flop

The improvement: Add gates to the S and R inputs so they only propagate when CLK = HIGH. This prevents unwanted state changes between clock pulses.

Timing behavior: When CLK = HIGH, S and R inputs directly affect the latch (same as asynchronous SR). When CLK = LOW, the latched state is frozen regardless of S/R inputs. This provides synchronization — the entire circuit only responds at predefined clock moments.

Key concept: Information can only change in sync with the clock. This is the foundation of synchronous digital design. Without this discipline, race conditions and metastability plague circuits.

Reading concern: Clocked SR still has the S=R=1 problem. The combinational logic feeding S and R must ensure this never happens. Many modern designs skip SR entirely and use D flip-flops instead, which are forced to have only one active input at a time.

D-Type Transparent Flip-Flop

Single data input D. Eliminates S=R=1 ambiguity.

While CLK=HIGH: Q follows D (transparent).

Jam Set (S) and Jam Reset (R) override D and CLK.

Problem: if D transitions during CLK=HIGH, output may be ambiguous.

Edge-Triggered D Flip-Flop

Why it matters: A transparent latch responds whenever CLK is HIGH — if D changes during that window, Q changes immediately, creating glitches. An edge-triggered flip-flop fixes this: it samples D only at one instant per clock cycle (the edge), then "closes the gate." D changes after the edge don't affect output until the next edge.

Design: Internally uses 2–3 SR stages in a master-slave configuration. The master SR latch captures D when CLK is one level, and the slave copies the master only at the opposite clock level.

Edge types:

  • Positive-edge (rising-edge): Triggers on LOW→HIGH transition. Most common in synchronous design (e.g., TTL 7474). Easier to visualize on scope captures.
  • Negative-edge (falling-edge): Triggers on HIGH→LOW transition. Used when the timing works better or for inverting pipelines.

Timing constraints (typical 5V TTL values):

  • tsetup ≈ 20ns — D must be stable and valid at least 20ns before the clock edge. Violate this and metastability can occur.
  • thold ≈ 3ns — D must remain stable for at least 3ns after the edge. If D toggles too quickly, the captured value is undefined.
  • tprop ≈ 10ns — Q appears (propagates) approximately 10ns after the clock edge. This delay chains in pipelines.

Reading concern: Setup and hold violations cause metastability — the flip-flop output oscillates for a brief time, settling to 0 or 1 unpredictably. At high speeds (>100MHz), matching clock and data line lengths on the PCB is critical to avoid violating thold. Many chip failures in production trace back to timing violations that weren't caught in simulation.

JK Flip-Flop

JKQnMeaning
101Set Q to 1
010Reset Q to 0
00Qn-1Hold previous state
11Qn-1Toggle (invert)

The "JK" magic: Unlike SR, when both J and K are 1, the JK toggles instead of entering an undefined state. This single feature enables divide-by-2 counters without extra logic. Connect Q̅ back to J and Q to K: on each clock edge, it inverts itself.

Practical use: JK is the "Swiss Army knife" of flip-flops. Every counter design textbook uses JK because J=K=1 is always safe and the toggle mode is powerful. Modern FPGAs implement primitives directly as D or SR, then synthesize JK behavior from them, but understanding JK thinking helps with digital design intuition.

Reading concern: The toggle mode (J=K=1) hides a race condition in transparent latches: if J and K are both 1 and CLK stays HIGH, Q toggles repeatedly, not just once. Edge-triggered JK flip-flops avoid this because they only sample at the edge. Always verify your datasheet specifies edge-triggered unless you want continuous toggling.

Embedded Timing Insight: At clock frequencies ≪100MHz, setup/hold/propagation delays are negligible. At modern microcontroller speeds (100MHz–1GHz+), these constraints dominate PCB layout decisions. Trace length differences between clock and data lines cause timing violations.

Chapter 4

Counters & Registers

Flip-flops chained together form registers (parallel memory) and counters (sequential state traversal) — fundamental to every embedded peripheral.

🔗 Registers → data storage in processors; Counters → timers, PWM, baud rate generation; Both are foundational to state machines (Section 7).

Divide-by-2 Counter

D flip-flop with Q connected back to D input.

Each clock edge toggles Q → output frequency = input / 2.

JK with J=K=1 achieves same result with no external wiring.

Embedded: clock prescalers, timer resolution scaling.

Asynchronous (Ripple) Counter

Structure: Chain flip-flops such that the output Q of one FF clocks the input of the next. FF₀ counts the input pulses and toggles at half the rate. FF₁ counts FF₀'s output and toggles at 1/4 the rate. And so on.

Example (4-bit): Input pulses clock FF₀ directly. FF₀'s Q output clocks FF₁. FF₁'s Q clocks FF₂. FF₂'s Q clocks FF₃. Result: a 4-bit binary counter (0→ 1→ 2→...15→ 0).

The critical problem — ripple delay: When you increment the counter, FF₀ toggles immediately. This change must propagate to FF₁ before FF₁ can toggle. Then FF₁'s change must reach FF₂, and so on. The MSB doesn't stabilize until 3–4 flip-flop delays have passed (roughly 30–40ns in old TTL logic). During this time, the intermediate states (0→ 8, then later 9→ 15) are visible briefly.

Why this is bad: If your logic decodes counter states (e.g., "when counter = 10, fire interrupt"), the ripple causes spurious intermediate decodings (e.g., 10→ 2→ 10 due to propagation delays). These glitches cause state machine errors or false triggers. At 100 MHz or faster, ripple delays become a significant fraction of a clock period.

Reading concern: Ripple counters are pedagogically useful but are rarely used in production circuits. Modern designs use synchronous counters where all FF's share the same clock and combinational logic predicts which bits toggle next, ensuring all bits change at the same time (within propagation delay of the combinational logic, not flip-flops). Synchronous is slower to design but much more reliable. Always avoid ripple counters in real embedded systems unless you have very slow clocks (<1 MHz) and no critical timing needs.

4-bit Data Register

4 D flip-flops sharing a single CLK line.

All 4 bits latch simultaneously on CLK edge — atomic parallel capture.

Embedded: capture SPI/parallel bus data, hold ADC results, store GPIO snapshots.

Shift Register

Structure: A chain of D flip-flops where the Q output of one FF feeds the D input of the next FF. All FFs share the same CLK.

Operation: On each clock edge, data "shifts" one position through the chain. If you input serial bits on the first FF's D input, after N clock cycles they appear in parallel at all N FF outputs.

Use cases:

  • Serial-to-Parallel conversion: 4 serial bits arrive one per clock cycle. After 4 clocks, all 4 bits are captured in parallel at Q₃Q₂Q₁Q₀. This is how UART receivers work: serial data bits are shifted in, and after 10 clocks (including start and stop bits), a complete byte is assembled.
  • Parallel-to-Serial conversion: Load all 4 bits simultaneously (using jam/async set/reset pins or a multiplexer). Then shift them out serially, one bit per clock. Used in SPI transmitters and UART transmitters.
  • Barrel shifting: Combine with multiplexers to rotate data by variable amounts, useful for bit-field extraction and DSP algorithms.

Embedded applications: SPI interface on any microcontroller is fundamentally a pair of shift registers (one RX, one TX). UART also uses shift registers internally. Many industrial communication protocols (CAN, I²S) rely on shift registers at the physical layer.

Reading concern: Shift register timing depends on the clock frequency. At very high speeds (>50 MHz), the capacitive loading of long shift chains can cause timing violations. Design tools must verify setup and hold times are met across all stages. Also, shift register behavior is sequential (N clocks for N bits), so low throughput compared to parallel buses, but uses only one or two signal lines, saving PCB routing.

Chapters 4 & 6

State Machines

State machines are clocked sequential processors: next state depends on current state + inputs. Every digital system from traffic lights to CPUs is a state machine.

🔗 State machines unify all previous concepts: flip-flops (memory), Boolean logic (next-state logic), K-maps (minimization). Replacing logic with ROM → microprocessor (Section 9).
The big picture: A state machine is the most general digital pattern. At each clock cycle, the machine reads current state and inputs, computes the next state using combinational logic, and transitions. Build a state machine for a traffic light. Scale it to handle 10 inputs and output 20 signals. Replace the logic gates with a lookup table (ROM). Feed the ROM an address computed from state + inputs, read back the next state and outputs. Now scale again: the ROM becomes program memory, the state register becomes a program counter, the inputs become instruction bits. This is a microprocessor. Every CPU in the world is a very large, very fast state machine executing a program stored in ROM.

General State Machine Structure

  • Data Register: n flip-flops storing present state.
  • Combinational Logic / ROM: computes next state from present state + k inputs.
  • Outputs: m bits derived from state (and possibly inputs).
  • CLK advances machine from present → next state.

Minimum bits needed for ℓ states: n = ⌈log₂(ℓ)⌉

Design Procedure

  1. Identify required states ℓ → determine bits n.
  2. Draw state diagram: circles = states, arrows = transitions, labels = inputs/outputs.
  3. Build truth table: n+k inputs, n+m outputs.
  4. Apply K-map or Boolean minimization to each output.
  5. Implement with flip-flops + gates (or ROM).
  6. Handle all unused states — give them an exit path (avoid "frozen" machine).

Synchronous vs Asynchronous

  • Synchronous: all FFs share one clock. Entire circuit changes state together. Predictable, glitch-free.
  • Asynchronous (ripple): FFs clocked by each other's outputs. Simple but slow — glitches at high speed.
  • All serious embedded designs use synchronous state machines.

Example: Up/Down 2-bit Counter

4 states (00, 01, 10, 11). 1 input u (u=1: count up, u=0: count down).

From truth table / K-map:

D₁ = u ⊕ (Q₀ ⊕ Q₁)   D₀ = Q₀

State diagram has 4 circles connected in a ring with directional arrows controlled by u.

Example: Divide-by-3 Counter

3 states needed → 2 bits (4 possible states). State D=11 is unused.

State sequence: A(00)→B(01)→C(10)→A. Output p=1 at state B only.

Unused state D must transition to A to prevent lockup.

Lockup = power-on enters unused state, machine never exits → firmware equivalent of infinite loop with no watchdog.

ROM-based State Machine

Replace combinational logic with a ROM lookup table.

Address = [present state bits | input bits]. Data word = [next state bits | output bits].

Example: divide-by-2-or-3 counter → 3-bit address (2 state + 1 input) → 8-location × 3-bit ROM.

Changing behavior = reprogramming ROM, not rewiring gates.

This IS the microprocessor model.

From State Machine → Microprocessor: With 8 input bits, machine can perform 2⁸ = 256 different operations per clock. Store the sequence of input bits in external memory (the program). Execute step-by-step. Add buses, registers, and I/O → full CPU. Every ARM Cortex-M in your embedded system is a state machine with millions of states.

Chapter 5

Analog/Digital Conversion

The interface between physical reality (analog) and digital computation. Every embedded sensor system depends on ADC quality and ADC/DAC selection.

🔗 ADC resolution links to binary number representation (Section 1). SAR ADC uses counters (Section 6) and comparator logic. DAC uses op-amp inverting configuration (power section concepts).
Why ADC/DAC matter: Real-world signals (temperature, sound, voltage) are analog — continuous. Digital processors work with bits — discrete values. ADCs convert analog → digital (sampling + quantization), and DACs convert digital → analog (lookup + filtering). The two key trade-offs: (1) resolution (how many bits?) — affects accuracy; (2) sampling rate (how many samples per second?) — must be ≥ 2× the highest frequency present, or aliasing destroys the signal. A microphone samples at 44kHz, a medical ECG at 1kHz, industrial vibration sensors at 10kHz. Choosing the wrong ADC for a job wastes power, misses fast events, or adds unnecessary latency.

A/D Resolution

  • Discrete levels: An n-bit ADC divides the input range into 2ⁿ equally-spaced quantization levels. Each level represents one unique binary code.
  • Resolution (LSB size): If the full range is 0–10V and you have 12 bits, the LSB = 10V / 2¹² = 10V / 4096 ≈ 2.44mV. Each ADC count change represents 2.44mV.
  • Accuracy vs resolution: Resolution is the theoretical smallest distinguishable voltage. Accuracy is the real-world error due to nonlinearity, offset, temperature drift, etc. A 12-bit ADC has 12-bit resolution but may only achieve 10-bit accuracy due to circuit imperfections.
  • Nyquist Theorem: To faithfully capture a signal with highest frequency fmax, sample at fsamp > 2·fmax. Audio (20kHz max) requires >40kHz (CD standard: 44.1kHz). Violate Nyquist and high frequencies "fold" into the audible range (aliasing), corrupting the signal irreversibly.
  • Anti-aliasing filter: In practice, place a low-pass filter before the ADC to remove all frequencies above fmax. Without it, noise and out-of-band signals alias into your measurement band and ruin data quality.

Reading concern: Resolution is not accuracy. A 16-bit ADC looks impressive on a datasheet but is useless if 4 of those bits are noise. Always read the "Signal-to-Noise Ratio (SNR)" and "Effective Number of Bits (ENOB)" columns in datasheets to know the real resolution. Also, quantization error is unavoidable: every conversion introduces ±0.5 LSB error. For 12-bit, 10V range: ±1.2mV error is inherent.

DAC — R-2R Ladder

Op-amp inverting config: Vout = −(Rf/Rin)·Vin.

Virtual ground: op-amp "−" input stays at 0V.

4-bit example: switches a₀–a₃ connect to 1.0V (HIGH) or GND (LOW).

Resistor values: R, 2R, 4R, 8R weight bits by their binary significance.

R-2R ladder advantage: only two resistor values needed → easier to fabricate with matching precision.

1010₂ (=10) → Itotal = 1/R + 1/4R = 5/4R → Vout = −5/4 V.

Flash ADC

2ⁿ−1 comparators in parallel, each with threshold Vth,i.

All compare simultaneously → fastest ADC type.

2-bit flash: 3 comparators, thresholds at 0.5V, 1.5V, 2.5V (for 0–3.5V range).

Resolution: ΔV/2ⁿ⁺¹.

Trade-off: doubles hardware per additional bit → impractical above ~8 bits. Used in oscilloscopes, RF receivers.

Successive Approximation (SAR) ADC

The algorithm: SAR uses a binary search to find the digital code that matches the input voltage. Start with the MSB set to 1; if the resulting DAC output is too high, clear that bit. Move to the next bit and repeat. In n clock cycles, you have the result.

Detailed example (8-bit, 0–10V, Vin=7.09V):

  1. Cycle 1: Try MSB=1 (code 10000000). DAC outputs 5V. Compare: 5V < 7.09V, so keep MSB=1.
  2. Cycle 2: Try next bit=1 (code 11000000). DAC outputs 7.5V. Compare: 7.5V > 7.09V, so clear bit. Code is now 10000000.
  3. Cycle 3: Try bit 5 (code 10100000). DAC outputs 6.25V. Compare: 6.25V < 7.09V, so keep. Code is now 10100000.
  4. ...continue for 5 more cycles...
  5. Final result (approximately): 10110100₂ = 180₁₀, which maps to 7.03V (close to input).

Speed and power: Conversion time is fixed at n clock cycles (8 clocks for 8-bit, 12 for 12-bit). No variable latency unlike flash. Power consumption is moderate. This is why almost every microcontroller (STM32, AVR, PIC, ESP32) includes a SAR ADC: good speed, good resolution, low power, low cost.

Typical specs: STM32L4 ADC: 12-bit resolution, 1 MSPS (1 million samples/sec), ±2.5% accuracy, <4µA in sleep mode.

Reading concern: SAR ADC requires an external holding capacitor on the input (acquisition capacitor) to stabilize the voltage during conversion. If you switch inputs rapidly between samples, charge redistribution errors (pedestal errors) degrade accuracy. Also, the internal DAC noise directly affects linearity, so supply bypassing is critical.

Single/Dual Slope ADC

Single slope: Charge capacitor from constant current source; count time until Vcap = Vin. Time ∝ Vin.

Dual slope: Charge capacitor proportional to Vin for fixed time τ, then discharge at fixed I₀; measure discharge time Δt = βVin.

Advantages: result independent of C value; integrates away 60Hz noise.

Use: multimeters, precision instruments. Slow but extremely accurate.

ADC Comparison for Embedded Design

TypeSpeedAccuracyCostBest For

Trade-off reasoning: Flash is simple (N comparators = N clock cycles) but wastes silicon and power on high-bit versions (256 comparators for 8-bit is a lot). SAR is clever: it reuses one DAC and comparator, so it scales well. Sigma-delta over-samples at 64–256× the target rate and uses digital filters to achieve extremely high accuracy; the oversampling noise-shapes quantization error out of the band of interest. Each has a place.

Reading concern: When choosing an ADC, consider: (1) Required resolution (8, 10, 12, 14, 16 bits?); (2) Maximum input frequency (how fast does your signal change?); (3) Required sampling rate (Nyquist theorem); (4) Power budget (battery-powered or bench?); (5) Cost and PCB space. Grabbing the "best" (sigma-delta) when SAR suffices wastes power and cost. SAR dominates because it's the "Goldilocks" solution for most embedded problems.

Chapter 7

Memories & Processors

Memory is organized random-access storage. ROM-based state machines become microprocessors — the culmination of all previous concepts.

🔗 Memory = registers (Section 6) + MUX/DEMUX (Section 4) + flip-flops (Section 5). ROM state machine (Section 7) generalizes to full processor.
The hierarchy and why it matters: A processor fetches instructions from program memory, stores temporary values in registers and cache, and can access slower main memory when needed. Each level is faster but smaller: registers (0.1ns) beat cache (1ns) beat DRAM (100ns) beat flash (10ms). An embedded designer chooses: put critical data in registers; let the CPU cache hot loops; spill to DRAM if needed. Understanding memory speed hierarchy is how you optimize firmware for real-time performance — it is the difference between a responsive embedded system and one that misses deadlines.

Memory Types

TypeReadWriteVolatileUse Case
SRAM✓✓YesCPU cache, registers, temp storage
DRAM✓✓Yes (refresh)System RAM (PCs, phones)
ROM✓✗NoFactory-programmed constants, boot code

Key concept: Volatility means the data is lost when power is removed. Non-volatile memory persists. The trade-off: volatile memory is faster and uses fewer transistors per bit, but your program and constants must live in non-volatile storage (flash) and be copied to volatile RAM at startup.

Reading concern: Don’t confuse "writable" with "reusable." PROM is writable but only once: fuses are blown and cannot be restored. EPROM is truly reusable but requires a UV eraser (not practical for field updates). Flash is electrically erasable and rewritable millions of times, making it the only practical choice for modern embedded systems.

SRAM vs DRAM

  • SRAM (Static RAM): Each bit is a flip-flop (or cross-coupled gates). Requires 4–6 transistors per bit. Data stays valid as long as power is on (no refresh needed). Fast: ~10ns access. Used for: CPU caches (L1, L2, L3), embedded scratch RAM, register files. Expensive per bit and large footprint, so limited to small amounts (kilobytes to low megabytes in embedded systems).
  • DRAM (Dynamic RAM): Each bit is a single transistor + capacitor. Capacitor holds a charge (1) or is empty (0), but charge leaks over milliseconds. Must refresh every ~1ms (read and re-write each row). Slower: ~100ns access. But much denser: ~1 transistor per bit. Used for: main system RAM in PCs, phones, servers. Cheap per bit, so practical for gigabytes. The refresh overhead is hidden by intelligent memory controllers that refresh in parallel with other operations.

Why the difference matters: You have a finite cache budget (SRAM is expensive). For every MB of SRAM, you lose 1-5MB of potential DRAM. Modern systems carefully balance: SRAM for hottest code/data (inner loops, critical data structures), DRAM for the rest. Firmware optimized for cache locality (keeping working set small) runs 2-3x faster than random-access firmware.

Reading concern: DRAM refresh and SRAM static power consumption both matter for battery-powered devices. In sleep mode, DRAM consumes little (refresh can be slowed). SRAM sips current continuously. For always-on IoT devices, total power includes standby power, not just active power. This is why datasheets list "typical power consumption" at various clock speeds and duty cycles.

RAM Organization

8×1 RAM architecture (1 byte, 8 locations):

  • Write path: Address bits (a₂a₁a₀) feed a DEMUX that selects one of 8 flip-flops. Data In is connected to the D inputs of all FFs. When WE (write enable) is asserted, only the selected FF latches Data In.
  • Read path: Address bits feed an 8-to-1 MUX that selects one FF output. That output feeds a 3-state (tri-state) buffer, which drives Data Out when OE (output enable) is asserted. Three-state logic allows multiple devices to share a bus: only the buffer with OE=1 drives the line; others are high-impedance (disconnected).

Key signals: Address (log₂N bits for N locations), Data In/Out (W bits for W-bit words), WE (write enable), OE (output enable), CS (chip select, not always shown), VSS/VDD (ground and power).

Access sequence: Stable address → wait for setup time → assert WE or OE → valid data appears after propagation delay → deassert WE/OE.

Reading concern: RAM timing must be considered in synchronous designs. If your state machine provides address at clock edge 0 but the RAM data doesn’t appear until edge 1+propagation delay, you may miss or fetch stale data. Always check timing diagrams in the datasheet. Also, the CS (chip select) signal is critical for multi-chip systems: only one chip should be active at a time to avoid bus conflicts.

Memory Capacity & Addressing

  • 1 bit = 1 binary digit. 1 byte = 8 bits.
  • 1K = 2¹⁰ = 1024 locations.
  • 8K×16 RAM: 8192 locations × 16-bit words = 128KB total.
  • External connections: 4 + log₂(n) for RAM vs 1 + 2n for standalone registers.
  • Access time ~10ns typical for SRAM.

From State Machine to Microprocessor

ROM-based state machine: PS = ROM address; ROM data = NS + outputs.

Add external inputs → combined address → 2ⁿ possible operations per state.

Store input sequence externally (program memory) → fetch next instruction each clock.

This IS the Harvard/Von Neumann architecture: data memory + program memory + ALU + registers + I/O buses.

Your STM32 MCU is this architecture implemented in 2nm+ CMOS with millions of states.

Embedded Memory Hierarchy

  • Registers: ~1 cycle. CPU-internal. Very limited.
  • Cache (SRAM): 1–10 cycles. On-chip. Kilobytes to megabytes.
  • Flash: 10–100 cycles (with cache). On/off chip. Program storage.
  • DRAM: 100–300 cycles. Off-chip. OS + heap.
  • eMMC/SD: Milliseconds. Mass storage.
